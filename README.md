1) What is the difference between null and undefined?
   Ans: undefined: যখন একটি ভেরিয়েবল ডিক্লেয়ার করা হয় কিন্তু কোনো মান (value) সেট করা হয় না, তখন সেটি ডিফল্টভাবে undefined থাকে।
   null: এটি একটি অ্যাসাইন করা মান, যা বোঝায় যে ভেরিয়েবলটি বর্তমানে ইচ্ছাকৃতভাবে খালি বা কোনো অবজেক্ট নেই।

2) What is the use of the map() function in JavaScript? How is it different from forEach()?
   Ans: map() একটি অ্যারেকে পরিবর্তন করে নতুন একটি অ্যারে তৈরি করতে ব্যবহৃত হয়।
   পার্থক্য: map() কাজ শেষে একটি নতুন অ্যারে রিটার্ন করে, কিন্তু forEach() কোনো কিছু রিটার্ন করে না, এটি শুধু লুপ চালিয়ে কাজ সম্পন্ন করে।
3) What is the difference between == and ===?
   Ans: == (Loose Equality): এটি শুধুমাত্র দুই পাশের মান (value) চেক করে, ডেটা টাইপ আলাদা হলেও সেটিকে কনভার্ট করে সমান করার চেষ্টা করে (যেমন: 5 == "5" সত্য)।
        === (Strict Equality): এটি মান (value) এবং ডেটা টাইপ (data type) উভয়ই চেক করে, তাই মান সমান হলেও টাইপ আলাদা হলে এটি মিথ্যা দেখায় (যেমন: 5 === "5" মিথ্যা)।
4) What is the significance of async/await in fetching API data?
   Ans: async/await জাভাস্ক্রিপ্টে অ্যাসিনক্রোনাস কাজকে (যেমন API থেকে ডেটা আনা) অনেক সহজ এবং দেখতে সাধারণ কোডের মতো করে তোলে।
   ১. এটি Callback Hell বা জটিল .then() চেইনিং কমিয়ে কোডকে পরিষ্কার ও পড়ার উপযোগী (Readable) করে।
   ২. এটি কোড এক্সিকিউশন থামিয়ে না রেখে ব্যাকগ্রাউন্ডে ডেটা লোড হতে সাহায্য করে এবং এর সাথে try/catch ব্যবহার করে খুব সহজে এরর হ্যান্ডেল করা যায়।
5) Explain the concept of Scope in JavaScript (Global, Function, Block).
   Ans:JavaScript-এ Scope নির্ধারণ করে একটি ভেরিয়েবল কোডের কোন অংশ থেকে অ্যাক্সেস করা যাবে। এটি তিন প্রকার:
       Global Scope: ভেরিয়েবলটি সব ফাংশনের বাইরে ডিক্লেয়ার করা হয় এবং কোডের যেকোনো জায়গা থেকে এটি ব্যবহার করা যায়।
       Function Scope: ভেরিয়েবলটি একটি নির্দিষ্ট ফাংশনের ভেতরে থাকে এবং শুধুমাত্র সেই ফাংশনের ভেতর থেকেই এটি অ্যাক্সেস করা সম্ভব।
       Block Scope: let বা const দিয়ে ডিক্লেয়ার করা ভেরিয়েবল যা শুধুমাত্র একটি নির্দিষ্ট ব্লকের ({ }) ভেতর কাজ করে (যেমন: if বা for লুপ)।
